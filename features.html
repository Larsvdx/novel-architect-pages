Features

A writing workspace built around the book, not a blank prompt box.

Novel Architect brings planning, drafting, revision, context, and review into one desktop application. AI features are optional, contextual, and designed to support the writer's decisions.

Library

Start from your books.

Manage a library of novels, open a book into a dedicated workspace, and keep book metadata, genres, descriptions, and the per-book AI behavior prompt in one place.

Chapters

Draft with structure and version history.

Use skeleton and story modes, reorder chapters, save explicit drafts, and keep chapter versions that preserve title, skeleton, story, and timestamp per snapshot.

Book output

Export when the manuscript is ready to share.

Export the full manuscript as a PDF from the desktop app, using the chapters and story text already stored in your local library.

AI as a tool, not the author

Three AI features that keep the writer in control.

Writing assistance

Ask for a chapter draft using the current chapter, selected previous chapters, active context, book description, genres, and your book-level AI instructions.

Revision assistance

Iterate on an existing chapter without losing the original. Generated revisions are saved as versions, so you choose what becomes part of the manuscript.

Reviewer

Request an editorial report for selected chapters, optionally with context and focus notes, then read saved reviews in the book workspace.

Context library

Continuity notes that help both you and the AI.

The context library is more than a pile of notes. It gives recurring story knowledge a predictable place, with structured forms for the details long-form fiction tends to lose track of.

Offline writing access

If the backend is unavailable, cached users and local-only profiles can still open local books, contexts, chapters, and versions. Backend-only features become available again when the app reconnects.

Simple desktop UI

The app uses a focused book workspace with tabs for Book, Chapters, Context, and Reviewer. Sidebars collapse when you need more writing room, and editing controls stay close to the active task.

Built-in spellcheck

Chapter stories, book fields, and context editors use a shared desktop spellcheck layer with red underlines and click-to-correct suggestions.

Story points for backend AI

AI requests use connected account features and story points. Cost previews and background job status keep the flow clear before and after a request is submitted.
