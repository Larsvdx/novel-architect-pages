Features

A workspace for the messy middle of writing a novel.

Novel Architect treats a novel as more than a document. It gives books, chapters, versions, context, reviews, and AI requests their own places so the work stays readable while it grows.

Library

Start from your books.

Keep every project in a searchable library, then open one book into a focused workspace with its own details, genres, description, and AI behavior prompt.

Chapters

Draft with structure and version history.

Draft the skeleton first, write the story when the shape is ready, reorder chapters, and keep version snapshots instead of treating every experiment as permanent.

Book output

Export when the manuscript is ready to share.

Export a complete manuscript to PDF from the desktop app when you want a clean read, a backup copy, or something easier to share.

AI as a tool, not the author

Three AI features that keep the writer in control.

Chapter drafting

Start with your skeleton. Choose the earlier chapters and context entries that should inform the draft, then submit the request as a background job.

Revision passes

Request another pass on saved prose without flattening your history. Each result is a version you can compare, edit, or ignore.

Editorial reviewer

Select chapters, add focus notes, and save the report beside the book so feedback does not disappear into a chat transcript.

Context library

Continuity notes that help both you and the AI.

The context library is where continuity stops being memory work. Use structured entries for the facts, rules, relationships, and atmosphere that should stay stable from scene to scene.

Offline writing access

If the backend is unavailable, cached users and local-only profiles can still open local books, contexts, chapters, and versions. Backend-only features become available again when the app reconnects.

Simple desktop UI

The app uses a focused book workspace with tabs for Book, Chapters, Context, and Reviewer. Sidebars collapse when you need more writing room, and editing controls stay close to the active task.

Built-in spellcheck

Chapter stories, book fields, and context editors use a shared desktop spellcheck layer with red underlines and click-to-correct suggestions.

Story points for backend AI

AI requests use connected account features and story points. Cost previews and background job status keep the flow clear before and after a request is submitted.
